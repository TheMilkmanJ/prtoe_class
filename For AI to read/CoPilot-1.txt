## CURRENT CODE STRUCTURE

prtoe_perturbations_derivs() Function:

1. Load PRTOE background quantities (phi, dphi, F, etc.)
2. Load PRTOE perturbation variables (delta_phi, Phi, eta, etc.)
3. Compute intermediate quantities:
   - Perturbed Ricci scalar delta_R
   - Effective mass m_eff2
   - Beta k² correction term
   - Metric sourcing
4. Evolve scalar field: d²δφ/dτ²
5. Compute stress-energy contributions:
   - Perturbed density δρ_prtoe
   - Perturbed pressure δp_prtoe
   - Momentum density θ_prtoe
6. Evolve metric potentials:
   - Bardeen potential Φ (from trace Einstein equation)
   - Slip η (from traceless Einstein equation)
7. Null limit: If xi ≤ 1e-8, explicitly zero all derivatives

Called From: perturbations_derivs() only when use_prtoe == TRUE

Result Variables Filled:
- dy[index_pt_delta_prtoe]
- dy[index_pt_ddelta_prtoe]
- dy[index_pt_Phi_prtoe]
- dy[index_pt_dPhi_prtoe]
- dy[index_pt_eta_prtoe]
- dy[index_pt_deta_prtoe]
- ppw->delta_rho (modified)

KEY TECHNICAL INSIGHTS
================================================================================

FLD vs PRTOE Comparison:

FLD (Fluid Dark Energy):
- 2 perturbation variables: δ (density), θ (velocity)
- Evolution: Simple coupled ODEs
- No metric evolution (uses Einstein equations)
- Gauge-independent formalism optional (PPF)
- Physics: Generic dark energy with equation of state w(a)
- Lines of code: 33 (extracted)

PRTOE (Piecewise Relativistic Tracker of Oscillations in Energy):
- 6 perturbation variables: δφ, dδφ, Φ_prtoe, dΦ_prtoe, η_prtoe, dη_prtoe
- Evolution: Complex coupled ODEs with metric coupling
- Metric evolution: Direct from DHOST-inspired equations
- Gauge: Newtonian gauge explicit
- Physics: Scalar field tracker with DHOST modifications
- Lines of code: 160 (extracted)

Key Realization:
- Einstein solver doesn't care HOW dark energy evolves (FLD vs PRTOE)
- Einstein solver only cares ABOUT: Total δρ_de, δp_de, ρ_de*θ_de
- FLD and PRTOE can evolve completely differently
- As long as they contribute correctly to totals, Einstein solver works
- This enables architectural separation while maintaining physics

Struct Scoping Lesson:
- Declaring "struct perturb_vector * pv = ppw->pv" in helper function fails
- Reason: Struct is forward-declared only in workspace context
- Solution: Access directly via ppw->pv-> instead of local pointer
- Pattern: Don't create local struct pointers in helper functions
- Better: Always use workspace pointers directly

Metric Variables Issue:
- metric_continuity, metric_euler are local variables in perturbations_derivs()
- Not part of workspace structure (ppw)
- FLD and other helpers need these values
- Solution: Pass as parameters to helper functions
- Alternative: Could add to workspace (would require larger refactoring)

## Critical Discovery: Multiple Unguarded PRTOE Index Reads

During investigation of the "PRTOE null limit integration stops early" issue, I discovered TWO critical buffer underflow vulnerabilities in background.c:

### Bug #1: Friedmann Equation PRTOE Variables (Line 815-859)
**Location:** background_functions(), Friedmann equation block
**Problem:** Lines 815-859 read PRTOE indices WITHOUT checking if they're valid
**Code:**
```c
if (pba->use_prtoe == _TRUE_) {  // ← OLD: No threshold check!
    double F = pvecback[pba->index_bg_F_prtoe];  // Reads -1 when null limit!
    double F_phi = pvecback[pba->index_bg_F_phi_prtoe];
    double phi_prime = pvecback[pba->index_bg_dphi_prtoe];
    // ... Modified Friedmann equation using these indices
}
```
**Impact:** When PRTOE is in null limit (xi_prtoe ≤ 1e-8), indices are set to -1. Reading pvecback[-1] causes buffer underflow, corrupting memory and potentially corrupting the entire background integration.

**Fix Applied:** Added proper threshold check:
```c
if (pba->use_prtoe == _TRUE_ && pba->xi_prtoe > 1e-8 && pba->Omega0_prtoe > 0.0) {
    // PRTOE Friedmann equation (only when physically active)
} else {
    // Standard Friedmann (for LCDM or null limit)
}
```

### Bug #2: PRTOE H-Dependent Quantities (Line 870-975)

**Problem:** Lines 870-975 read PRTOE background table indices WITHOUT validation

if (pba->use_prtoe == _TRUE_) {  // ← OLD: No threshold check!
